# VISION — AERIS (THE SUPREME ORCHESTRATOR)

## 1. Das Ziel (State of the Art 2026)
Erschaffung eines "Agentic OS" für die Softwareentwicklung, das im Terminal (Textual/Zellij) lebt, die UX einer High-End-Desktop-App bietet und durch Voice (STT/TTS) gesteuert wird. AERIS fungiert als **Supreme Orchestrator**, der ein Team von spezialisierten KI-Experten führt.

## 2. Architektonische Schichten (Die Fähigkeiten)

### 2.1 Die AERIS FACTORY (Universal UI-Engine)
AERIS besitzt die Fähigkeit, als **Frontend-Designer** für sich selbst und alle zukünftigen Projekte zu fungieren.
- **Blueprint-System:** UI-Strukturen werden in einfachen Konfigurationsdateien (YAML/JSON) definiert. Jannis kann Positionen, Texte und Elemente ohne Programmierung hinzufügen, löschen oder verschieben.
- **Palette-System:** Visuelle Styles (Farben, Abstände, Rahmen) werden in TCSS-Dateien definiert.
- **Live-Designer-Mode:** Echtzeit-Vorschau aller Änderungen. AERIS aktualisiert sein Gesicht in Millisekunden, sobald Jannis den Blueprint oder die Palette ändert (Elementor-Style).
- **Presets & Customization:** Auswahl aus professionellen "Architect-Presets" oder vollständige manuelle Kontrolle.

### 2.2 Orchestrierungs-Schicht (The Supreme Orchestrator)
- **Task-Decomposition:** Zerlegung komplexer Ziele in atomare "KI-Häppchen". Keine "One-Shot" Fehler mehr.
- **Expert Swarm Manager:** Dynamisches Spawnen von Experten (Coder, Tester, Dokumentar).
- **Validation-Loop:** Die "Cascade" prüft Ergebnisse via Terminal-Monitoring, bevor der nächste Schritt freigegeben wird.

### 2.3 Intelligenz & Memory-Schicht (The Brain)
- **Universal Model Bridge:** Wechsel zwischen Gemini, Claude, GPT und GitHub Models.
- **Tri-Tier Memory:** Working Memory (Speed), Episodic Memory (LanceDB Vektor-Store), Semantic Memory (Knowledge Graph).

### 2.4 Sicherheits-Schicht (The Agent-Zero Sandbox)
- **Docker-First Execution:** Agenten agieren in isolierten Containern.
- **Dynamic Tooling:** Agenten schreiben sich zur Laufzeit eigene Python-Skripte in der Sandbox.

## 3. Daten-Struktur
- `~/.config/aeris/`: Globale Konfiguration, Vault für API-Keys/Tokens.
- `.aeris/` (Projekt-Root): Projekt-Gedächtnis, Task-Logs, Blueprints und Paletten.
